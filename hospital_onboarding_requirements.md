# 병원 입점 신청 데이터 요구사항

`Hospital` 모델 분석을 바탕으로, 병원 입점 랜딩 페이지의 신청 폼에서 받아야 할 정보들을 정리했습니다. (한국어 입력 기준)

## 1. 필수 정보 (입점 신청 및 검증용)
`Hospital` 레코드를 생성하고 병원의 실재 여부를 확인하기 위해 반드시 필요한 정보들입니다. 

| 필드명 | 설명 | 데이터베이스 컬럼 | 타입 | 검증 필요사항 |
| :--- | :--- | :--- | :--- | :--- |
| **병원명** | 병원 공식 명칭 (한글) | `name` | 텍스트 | 필수 |
| **대표자명** | 사업자 등록증 상 대표자 | `representative_name` | 문자열 | 필수 |
| **사업자등록번호** | 병원 검증용 | `business_registration_number` | 문자열 | 10자리 숫자 |
| **전화번호** | 대표 연락처 | `phone` | 문자열 | XX-XXXX-XXXX 또는 XXX-XXXX-XXXX |
| **주소 (구/군)** | 위치 필터링용 (서울시 00구) | `district` | Enum | 유효한 지역 선택 |
| **주소 (전체)** | 상세 주소 (한글) | `address` | 텍스트 | 필수 |
| **이메일** | 연락용 대표 이메일 | `email` | 문자열 | 이메일 형식 |

## 2. 프로필 정보 (신청 시 또는 추후 수집 가능)
병원 프로필을 풍부하게 만드는 정보들로, 입점 승인 후 입력하게 하거나 신청 시 선택적으로 받을 수 있습니다.

| 필드명 | 설명 | 데이터베이스 컬럼 | 타입 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **병원 소개** | 간단한 소개글 (한글) | `description` | 텍스트 | |
| **웹사이트** | 공식 홈페이지 | `website` | URL | 선택 사항 |
| **진료 시간** | 운영 시간 정보 | `hospital_schedules` | Nested | 초기엔 텍스트로 간단히 받을 수 있음 |

## 3. 이미지 및 파일
병원의 시각적 정보를 위한 업로드 항목입니다.

| 필드명 | 설명 | 데이터베이스 관계 | 타입 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **병원 로고** | 브랜드 로고 이미지 | `logo_image` (has_one) | 파일(Image) | |
| **원내 이미지** | 병원 내부 전경 등 | `gallery_images` (has_many) | 파일(Image) | 다중 선택 가능 |

## 4. 계정 / 관리자 정보
병원 관리자용 초기 `AdminUser` 생성을 위해 필요한 정보입니다.

| 필드명 | 설명 | 데이터베이스 컬럼 | 타입 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **담당자명** | 대표자와 다를 경우 | `admin_users.name` | 문자열 | |
| **담당자 연락처** | 직통 번호 | `admin_users.phone` | 문자열 | |
| **담당자 이메일** | 로그인 ID로 사용 | `admin_users.email` | 문자열 | 실제 로그인에 사용됨 |

## 5. 참고사항
- **스키마 정리 완료**: 이전 분석에서 발견된 `registration_step` 및 `bank_account_info` 관련 불일치 코드는 정리되었습니다.
- **다국어 입력**: 입력 폼에서는 **한국어**만 입력받으면 됩니다.
